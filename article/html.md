### **一、HTML 标签全景与语义化核心**

#### **1. 文档骨架标签**
这是每个页面的基础框架。

| 标签                  | 作用与说明                                                                                              | 现代实践                                       |
| :-------------------- | :------------------------------------------------------------------------------------------------------ | :--------------------------------------------- |
| `<!DOCTYPE html>`     | **文档类型声明**，告诉浏览器使用 HTML5 标准解析。必须放在第一行。                                       | 必须。固定写法。                               |
| `<html lang="zh-CN">` | **根元素**，包裹所有页面内容。`lang` 属性**极其重要**，指定页面主语言，关乎浏览器翻译、屏幕阅读器发音。 | 必须。`lang` 值务必准确（如英文页面用 `en`）。 |
| `<head>`              | **文档头**，存放元信息（`meta`）、标题、引入外部资源链接等，**不显示在页面中**。                        | 必须。                                         |
| `<body>`              | **文档体**，存放所有可见的页面内容。                                                                    | 必须。                                         |

#### **2. 元信息标签 (`<head>` 内)**
控制页面如何被浏览器、搜索引擎等处理。

| 标签                         | 作用与说明                                                                                                                   |
| :--------------------------- | :--------------------------------------------------------------------------------------------------------------------------- |
| `<title>`                    | **页面标题**，显示在浏览器标签页、搜索引擎结果中。应简洁、包含关键词。                                                       |
| `<meta charset="UTF-8">`     | **字符编码声明**，防止乱码。必须。                                                                                           |
| `<meta name="viewport" ...>` | **视口设置**，控制移动端缩放和布局。**响应式必备**：`<meta name="viewport" content="width=device-width, initial-scale=1.0">` |
| `<link>`                     | **引入外部资源**，如 CSS 文件（`rel=“stylesheet”`）、网站图标（`rel=“icon”`）。                                              |
| `<style>`                    | **内联 CSS 样式**。                                                                                                          |
| `<script>`                   | **引入或编写 JS 代码**。通常放在 `</body>` 前，或使用 `defer`/`async` 属性。                                                 |

#### **3. 文本与内容结构化标签**
从通用容器到语义化分区。

| 标签                | 作用与说明                                                                                           | **与通用 `<div>` 的关键区别**                         |
| :------------------ | :--------------------------------------------------------------------------------------------------- | :---------------------------------------------------- |
| **`<div>`**         | **通用内容容器**，无语义。用于样式布局或脚本操作。                                                   | **基准标签**。当没有更合适的语义标签时使用。          |
| **`<span>`**        | **通用行内容器**，无语义。用于包裹少量文字或行内元素。                                               | 行内版的 `<div>`。                                    |
| **`<p>`**           | **段落**。浏览器会自动为其添加上下边距。                                                             | 用于文本段落，而非仅仅为了换行。                      |
| **`<h1>` ~ `<h6>`** | **标题**，重要性递减。`<h1>` 通常一个页面只用一次，作为主标题。                                      | 建立文档大纲，**勿仅用于调整文字大小**。              |
| **`<main>`**        | **主内容区**。每个页面应**只有一个**，包含页面核心内容。                                             | 比 `<div id=“main”>` 语义明确，帮助辅助工具快速定位。 |
| **`<header>`**      | **页头**。通常包含网站标识、导航。**不仅用于页面顶部**，也可用于 `<article>` 或 `<section>` 的头部。 | 比 `<div class=“header”>` 语义明确。                  |
| **`<footer>`**      | **页脚**。通常包含版权、联系方式等。同 `<header>`，可用于区块底部。                                  | 比 `<div class=“footer”>` 语义明确。                  |
| **`<nav>`**         | **导航区域**。用于包含一组主要的导航链接。                                                           | **务必替换**你的 `<div class=“nav”>`。                |
| **`<article>`**     | **独立可分发的内容**。如博客文章、新闻、论坛帖子。应能独立于页面其他部分存在。                       | 用于自成一体的内容区块。                              |
| **`<section>`**     | **文档中的主题性分组**。通常带有标题，用于对内容分章分节。                                           | 比 `<div>` 更有结构语义，但不如 `<article>` 独立。    |
| **`<aside>`**       | **侧边栏或附属信息**。与周围内容间接相关，如引用、侧栏、广告。                                       | 用于非主体、补充性的内容。                            |
| **`<address>`**     | **联系信息**。用于提供与最近 `<article>` 或 `<body>` 相关的联系信息。                                | 不仅用于“地址”，可以包含邮箱、电话等。                |

#### **4. 列表标签**
| 标签   | 作用与说明                                       | 注意                                              |
| :----- | :----------------------------------------------- | :------------------------------------------------ |
| `<ul>` | **无序列表**，项目间无顺序关系。                 | 默认带项目符号（如圆点）。                        |
| `<ol>` | **有序列表**，项目间有顺序或步骤关系。           | 默认带数字序号。可通过 `type`、`start` 属性控制。 |
| `<li>` | **列表项**，必须放在 `<ul>` 或 `<ol>` 内部。     |                                                   |
| `<dl>` | **描述列表**。用于定义术语及其描述（如词汇表）。 | 现代语义化的重要标签。                            |
| `<dt>` | **描述术语**，放在 `<dl>` 内。                   |                                                   |
| `<dd>` | **术语描述**，放在 `<dl>` 内，紧跟 `<dt>`。      |                                                   |

#### **5. 文本语义标签**
用于给文字赋予特定含义，**切勿仅用于样式**。

| 标签           | 作用与说明                                                           | **与旧标签的对比与选择**               |
| :------------- | :------------------------------------------------------------------- | :------------------------------------- |
| `<em>`         | **强调**。改变句子含义（如“我**是**对的”）。通常斜体，但本质是语义。 | **优先于** `<i>`。用于语义强调。       |
| `<strong>`     | **重要**。表示内容的高度重要性、严重性或紧迫性。通常加粗。           | **优先于** `<b>`。用于语义上的重要。   |
| `<i>`          | **技术术语、外来语**或**语气变化**。无强调含义。                     | 仅用于**无强调**的样式差异。           |
| `<b>`          | **引起注意**但不提升重要性，如摘要中的关键词。                       | 仅用于**无重要性**的样式差异。         |
| `<mark>`       | **高亮标记**，用于引用或关注目的。如搜索关键词。                     | 自带黄色背景，语义明确。               |
| `<time>`       | **时间/日期**。`datetime` 属性提供机器可读格式。                     | **强烈推荐**，便于搜索引擎和程序理解。 |
| `<code>`       | **内联代码片段**。等宽字体。                                         |                                        |
| `<pre>`        | **预格式化文本**，保留空格和换行。用于多行代码块。                   | 常与 `<code>` 嵌套使用。               |
| `<blockquote>` | **块级引用**，通常有缩进。`cite` 属性可指明来源 URL。                | 用于引用大段他处内容。                 |
| `<cite>`       | **引用来源的标题**（如书名、文章名）。                               |                                        |

#### **6. 媒体与嵌入标签**
| 标签        | 作用与说明                                                                       | **关键属性**                                                                                                   |
| :---------- | :------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------- |
| `<img>`     | **嵌入图片**。                                                                   | `src`（源）, `alt`（替代文本，**必须**）, `width`/`height`（建议设置，防布局偏移）, `loading=“lazy”`（懒加载） |
| `<picture>` | **响应式图片容器**，内部可包含多个 `<source>` 和 `<img>`，让浏览器根据条件选择。 | 用于艺术指导（不同尺寸/裁剪）或下一代格式（如 WebP）回退。                                                     |
| `<audio>`   | **嵌入音频**。                                                                   | `controls`（显示控件）, `src`, `autoplay`（慎用）, `loop`                                                      |
| `<video>`   | **嵌入视频**。                                                                   | `controls`, `src`, `width`/`height`, `poster`（封面图）                                                        |
| `<iframe>`  | **嵌入另一个网页**。                                                             | `src`, `title`（**必须**，用于可访问性）, `sandbox`（安全限制）                                                |
| `<canvas>`  | **通过 JS 绘制图形（图表、游戏）**。                                             | 只是一个画布，内容全靠 JS 绘制。                                                                               |
| `<svg>`     | **可缩放矢量图形**。                                                             | 直接内嵌 XML 格式的矢量图形代码。                                                                              |

#### **7. 表单与交互标签**
| 标签         | 作用与说明                                             | **关键属性**                                                                                                                                             |
| :----------- | :----------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<form>`     | **表单容器**，用于收集用户输入。                       | `action`（提交地址）, `method`（GET/POST）                                                                                                               |
| `<input>`    | **多功能输入控件**，类型由 `type` 决定。               | `type`（`text`, `password`, `email`, `number`, `date`, `checkbox`, `radio`, `file` 等）, `name`（提交的键名）, `placeholder`（提示）, `required`（必填） |
| `<textarea>` | **多行文本输入**。                                     | `rows`, `cols`, `wrap`                                                                                                                                   |
| `<select>`   | **下拉选择框**，内部包含 `<option>`。                  |                                                                                                                                                          |
| `<option>`   | **下拉选项**。                                         | `value`（提交的值）, `selected`（默认选中）                                                                                                              |
| `<label>`    | **表单标签**，与表单控件关联。**点击标签可聚焦控件**。 | `for`（关联控件的 `id`），或包裹控件。**务必使用**，提升可用性和可访问性。                                                                               |
| `<button>`   | **按钮**。                                             | `type`（`submit`/`reset`/`button`，默认为 `submit` 需注意），**优先于** `<input type=“button”>`                                                          |
| `<fieldset>` | **表单控件组**，用于语义和样式分组。                   |                                                                                                                                                          |
| `<legend>`   | **`<fieldset>` 的标题**。                              |                                                                                                                                                          |

#### **8. 表格标签（仅用于真实数据表格）**
| 标签                            | 作用与说明                                                              |
| :------------------------------ | :---------------------------------------------------------------------- |
| `<table>`                       | **定义表格**。                                                          |
| `<caption>`                     | **表格标题**。                                                          |
| `<thead>`、`<tbody>`、`<tfoot>` | **表格头、体、脚**，用于分组。                                          |
| `<tr>`                          | **表格行**。                                                            |
| `<th>`                          | **表头单元格**，通常加粗居中。`scope` 属性（`row`/`col`）提升可访问性。 |
| `<td>`                          | **数据单元格**。                                                        |

### **二、HTML 核心属性详解**

#### **1. 全局属性（可用于几乎所有标签）**
| 属性              | 作用与说明                                                                            |
| :---------------- | :------------------------------------------------------------------------------------ |
| `id`              | **唯一标识符**。全页面唯一，用于锚点链接或 JS/CSS 精准选择。                          |
| `class`           | **类名**。可重复，用于 CSS 样式和 JS 选择的主要方式。                                 |
| `style`           | **内联样式**。优先级高，但不利于维护，**慎用**。                                      |
| `title`           | **额外提示信息**。鼠标悬停时显示。                                                    |
| `data-*`          | **自定义数据属性**。用于存储与元素相关的私有数据，供 JS 读取。如 `data-id=“123”`。    |
| `hidden`          | **布尔属性**，隐藏元素（`display: none`）。                                           |
| `tabindex`        | **控制 Tab 键导航顺序**。`0` 表示可聚焦，`-1` 表示不可通过 Tab 聚焦但可通过 JS 聚焦。 |
| `contenteditable` | **使元素内容可编辑**。                                                                |
| `draggable`       | **使元素可拖动**，用于拖放 API。                                                      |

#### **2. 关键专有属性**
| 属性（所属标签）                                      | 作用与说明                                                                                                                                                                                                                                                                                                                                                                       |
| :---------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `href` (`<a>`)                                        | **链接目标 URL**。可以是外部网址、内部路径（`/page`）、锚点（`#section`）或协议（`mailto:`）。                                                                                                                                                                                                                                                                                   |
| `src` (`<img>`, `<script>`, `<iframe>`)               | **嵌入资源的来源 URL**。                                                                                                                                                                                                                                                                                                                                                         |
| `alt` (`<img>`, `<area>`)                             | **图片无法显示时的替代文本**。对可访问性**至关重要**。空字符串 (`alt=“”`) 表示装饰性图片。                                                                                                                                                                                                                                                                                       |
| `target` (`<a>`, `<form>`)                            | **如何打开链接**。`_blank`（新标签页）, `_self`（当前页，默认）。                                                                                                                                                                                                                                                                                                                |
| `rel` (`<a>`, `<link>`)                               | **与目标资源的关系**。重要值：`noopener`（安全，与 `_blank` 同用）, `noreferrer`, `stylesheet`, `icon`。                                                                                                                                                                                                                                                                         |
| `type` (`<input>`, `<button>`, `<script>`, `<style>`) | 指定类型。对于 `<input>` 尤其关键（`email`, `tel` 会触发移动端特定键盘）。                                                                                                                                                                                                                                                                                                       |
| `for` (`<label>`)                                     | **关联到表单控件的 `id`**，实现点击标签即聚焦控件。                                                                                                                                                                                                                                                                                                                              |
| `aria-*` (**任何标签**)                               | **可访问性富互联网应用属性**。为屏幕阅读器等辅助技术提供语义。核心属性：<br>• `aria-label`：直接提供元素描述。<br>• `aria-labelledby`：指向另一个为其提供标签的元素的 `id`。<br>• `aria-describedby`：指向提供详细描述的元素 `id`。<br>• `aria-hidden=“true”`：对辅助技术隐藏。<br>• `role`：定义元素的语义角色（如 `role=“button”`, `role=“navigation”`），在自定义组件中常用。 |
| `loading` (`<img>`, `<iframe>`)                       | **懒加载**。设为 `lazy` 可延迟加载位于可视区域外的图片/框架。**性能优化利器**。                                                                                                                                                                                                                                                                                                  |

### **三、总结：你的 HTML 升级行动清单**

1.  **拥抱语义化容器**：立即用 `<nav>`、`<main>`、`<header>`、`<footer>`、`<section>`、`<article>` 替换相应的 `<div>`。
2.  **使用正确的交互元素**：用 `<button>` 替换 `<input type=“button”>`。用 `<ul><li>` 重构你的导航链接。
3.  **完善媒体与链接**：
    *   为所有 `<img>` 提供有意义的 `alt` 描述。
    *   为所有 `<a>` 和 `<button>` 提供清晰的文本或 `aria-label`。
    *   考虑为轮播图 `<img>` 添加 `loading=“lazy”`（除第一张）。
4.  **引入可访问性基础**：
    *   为图标按钮、表单控件添加 `aria-label`。
    *   为自定义组件（如轮播图）添加 `role` 和 `aria-*` 属性。
    *   确保表单控件有对应的 `<label>`。
5.  **优化文档结构**：
    *   将页面内容包裹在 `<main>` 中。
    *   使用 `<h1>`-`<h6>` 建立清晰的标题层级。
    *   考虑使用 `<time>` 标记日期时间。

这些改变将使你的代码从“能显示”升级为**结构清晰、对机器友好、对所有用户可访问的现代 HTML**。这是你迈向专业前端开发的重要一步。
